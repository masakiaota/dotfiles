# コミットメッセージの例

Conventional Commits準拠の日本語コミットメッセージの詳細な例集。

## 新機能追加 (feat:)

```
feat: インテリジェントドキュメント分類システムに感情分析機能を追加

- 新しい感情分析モデルを推論エンジンに統合
- データモデル定義に感情分析用のレスポンス形式を定義
- 評価用のテストケースとサンプルデータを追加
```

## バグ修正 (fix:)

```
fix: ユーザビリティ評価APIのレスポンス形式を修正

- JSONスキーマの型定義エラーを修正
- 評価スコアが負の値になる問題を解決
- エラーハンドリングで適切なステータスコードを返すように変更
```

## リファクタリング (refactor:)

```
refactor: 共通ユーティリティをutil/data_processor.pyに統合

- 各タスクで重複していたデータ処理ロジックを共通化
- 型ヒントを追加してコードの可読性を向上
- 不要な依存関係を削除してDockerイメージサイズを削減
```

## テスト追加 (test:)

```
test: 検索結果最適化エンジンのテストカバレッジを拡充

- エッジケースのテストケースを追加
- APIレスポンスの型検証テストを実装
- パフォーマンステスト用のベンチマークデータを作成
```

## ドキュメント更新 (docs:)

```
docs: 品質管理システムのREADMEを更新

- 新しいテストケースの実行方法を追記
- 各品質指標の計算方法を詳細化
- トラブルシューティングセクションを追加
```

## 依存関係更新 (chore:)

```
chore: パッケージマネージャーを使用してPython依存関係を更新

- セキュリティ脆弱性のあるパッケージを最新版に更新
- 非互換性のある変更点をコードに反映
- 全てのタスクでテストが正常に動作することを確認
```

## パフォーマンス改善 (perf:)

```
perf: 画像認識処理のバッチ処理効率を改善

- モデル推論のバッチサイズを最適化
- 不要なメモリコピーを削減
- 推論時間を約30%短縮
```

## セキュリティ修正 (security:)

```
security: APIキーの漏洩リスクを軽減

- ハードコードされた認証情報を環境変数に移行
- ログ出力からセンシティブな情報を除外
- 入力検証を強化してインジェクション攻撃を防止
```

## スタイル修正 (style:)

```
style: Black フォーマッターによるコード整形

- 行長の制限を80文字に統一
- インポート文の順序を整理
- 余分な空行を削除
```

## タイプ選択のヒント

| 変更内容 | 推奨タイプ |
|----------|------------|
| 新しい機能やエンドポイントの追加 | `feat:` |
| エラーや不具合の修正 | `fix:` |
| コードの構造改善（動作は同じ） | `refactor:` |
| 設定ファイルや依存関係の更新 | `chore:` |
| README、コメント、APIドキュメント | `docs:` |
| テストコードの追加・修正 | `test:` |
| 処理速度やメモリ使用量の改善 | `perf:` |
| セキュリティ関連の修正 | `security:` |
| フォーマッター実行、空白調整 | `style:` |
